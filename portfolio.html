LQDOJ (Le Quy Don Online Judge)

Description

LQDOJ is an automated code judging platform developed based on the open-source DMOJ framework. Originally established to provide a learning environment specifically for computer science students at Le Quy Don Specialized High School in Vietnam, LQDOJ has expanded registration to the public, becoming a large playground for the entire community of computer science enthusiasts.

Contribution

I developed a frequency-based plagiarism detection system within the LQDOJ platform. My role extended to analyzing a vast dataset comprising 2.9 million programming submissions. By implementing the Aho-Corasick algorithm for efficient pattern matching, I was able to reduce the analysis runtime by an impressive 70% compared to previous methods. The project resulted in a significant 60% reduction in cheating instances, maintaining the platform's integrity and promoting a fair competitive environment.

In addition, I enhanced the Chat feature of the platform. I improved the messaging experience for users by developing an algorithm for detecting spam and validating messages. Besides coding the backend, I also developed the UI for the Chat, creating a text-wrapping chat box and designing buttons for file uploads and message submissions.

Technologies

For the backend development and database interactions of my project, I employed Django, a high-level Python web framework. I implemented the Aho-Corasick algorithm in C++ for efficient pattern matching. This algorithm was integrated with MariaDB to handle and evaluate the vast number of user submissions efficiently. For the frontend, I used HTML/CSS and JavaScript to create a user-friendly interface.

Connect Four

Description

This project, developed for CSCE 314, is a Java and JavaFX implementation of the Connect Four Game where a player competes against an artificial intelligence (AI) opponent. The game has been designed to offer a variety of options, including the ability to choose who initiates the game, the selection of the AI's difficulty level, and the functionality to save and load the game state as well as a log of moves.

Contribution

I reimagined the classic Connect Four game by creating a Java and JavaFX-based application that pits a human player against an AI opponent. My main objective was to provide a robust and interactive gaming experience that reflects the strategic depth of the traditional board game. I implemented an intuitive user interface that allows players to choose who starts first and select the AI's difficulty level, catering to both beginners and experienced users.

In order to enhance the user experience, I implemented functionality enabling players to save and load game states and move logs, allowing them to review past games and continue playing without lost progress. Leveraging the data from saved games, I plan to apply machine learning techniques to enhance the AI, aiming to create an opponent that is both challenging and adaptive to the player's strategies, making the game more engaging and replayable.

Technologies

The project was primarily built in Java, taking advantage of the language's robustness and object-oriented capabilities to structure the game's core logic. I developed two distinct AIs using different algorithms: a Random Algorithm and the Minimax Algorithm. JavaFX was utilized to create dynamic and visually appealing user interfaces. CSS was additionally used to improve the aesthetic appeal of the user interface.
